> 小记一下，在此之前一直是采用 github 托管 hexo 博客，但是确实是访问速度太慢了，便想着优化一下博客网站的搭建方式，正好今天学有所成，便来显摆(\^V\^)

# 选择博客类型

网上有许多类型的博客，我比较省钱，不想买服务器，就选择了 hexo+github page 托管的方式进行搭建，后来发现 github page 不挂梯子速度太慢了，查了相关资料选择了 netlify 托管，free 账户每个月 300 额度，相当于每个月可以上传 20 次博客，但是访问另算，所以还是要少上传，访问的话我本身不打算博客有多少人访问，纯自己写写；其他博客的类型嘛，回头补充吧--2025.12.18

# 搭建方式

省流版：

1. 配置本地环境(Node.js,Git)
2. 安装 hexo
3. github page 或者 netlify 托管

详细版：

因为我只试过这一种搭建方式，就便只说这一种 **(其实 AI 这么发达，随便搜一下也能查到)**

首先需要安装本地环境，hexo 是基于 Node.js 生成的，所以需要安装 Node.js 工具，版本的话我的是 v24.10.0，npm 版本是 11.6.1，npm 应该是安装 Node.js 时附带的

这个安装完成后，需要再安装一个 git 工具用来博客版本控制，方便管理远程仓库，这个官网下载安装即可

其次下载一个文档编辑器，可以用 typora，也可以用 VSCode，我用的是 VSCode，因为可以使用 git 插件，不用手动通过命令行同步

在 github 构建一个仓库，仓库名为`账号.github.io`，至少我是这样的

这些都弄好之后，在 VSCode 打开 cmd 控制台，输入

```bash
npm install -g hexo-cli
```

选择一个文件夹作为博客根目录，后续的写博客工作就在这个目录下进行

```bash
hexo init
```

安装依赖包

```bash
npm install
```

之后该目录下面会生成 hexo 的文件，你可以从网上下载主题或者根据文档对 hexo 博客进行修改
[hexo-themes](https://hexo.io/themes/)不止这一个网站有主题，可以试着自己找找

接下来是上传到 github，首先在 github 创建一个仓库

1. github page
   名字必须为`用户名.github.io`，其他无所谓，在仓库的 Settings-Pages-Source 里选择 Master 分支，保存后 Github 会生成访问地址，设置完成后，需要给 hexo 设置 git 插件

```bash
npm install hexo-deployer-git --save
```

编辑 hexo 配置文件\_config.yml，修改 deploy 部分

```yaml
deploy:
  type: git
  repo: https://github.com/你的用户名/你的用户名.github.io.git
  branch: master
```

觉得慢可以使用 ssh，但是需要生成 ssh 密钥，这个网上教程很多

```yaml
deploy:
  type: git
  repo: git@github.com:账号/账号.github.io.git
  branch: master
```

2. 托管

这个我熟啊，两种托管方式，一种是通过 github actions，设置管道，每次仓库更新时，自动触发逻辑处理，最终还是通过 github page 展示，启动速度慢的问题还是没解决，所以着重讲第二种方式

- netlify 托管

去 netlify 官网绑定 github 账号，选择创建好的仓库，在 Build-Settings 里设置
Build command : `hexo clean && hexo generate`
Publish directory: `public`

之后仓库更新时，netlify 会自动更新静态网页

# 操作步骤

在本地控制台输入

```bash
hexo clean # 清除旧静态文件
hexo g  # 生成静态文件（g = generate）
hexo s  # 启动本地服务器（s = server）
hexo d # 部署到服务器 (d = deploy)
```
