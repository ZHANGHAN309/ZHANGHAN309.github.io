概述 — 芯片厂 FAE（软件方向）

本笔记针对希望从 0 基础转行到“芯片厂 FAE（软件方向）”的人制定，重点在软件层面的能力：固件与 boot 流程、BSP/驱动、内核与用户态接口、SDK/示例代码、性能/功耗优化、软件测试与自动化、OTA 与版本管理、客户 SDK 支持与问题定位。计划为 6 个月（24 周），每项包含可量化考核目标与交付物。

核心能力矩阵（软件知识点）

- 工具链与构建：交叉编译器、Make/CMake、交叉工具链配置、构建系统（yocto、buildroot 可选）。
- 引导与固件：bootloader（U-Boot 或厂商自定义）、Flash/烧写流程、Boot log 分析、CRC/签名校验、secure boot 概念。
- BSP 与驱动开发：Platform/SoC 初始化、设备树（Device Tree）、内核模块与字符/平台/网络驱动、DMA/IRQ 机制、内存映射 I/O。
- 系统集成与用户态 API：SDK 设计、REST/IPC/Netlink/IOCTL 等接口约定、示例程序与文档。
- 调试与诊断：GDB、kgdb、串口日志、core dump 分析、跟踪工具（ftrace、perf）、日志等级与日志收集策略。
- 性能与功耗优化：性能剖析、热点定位、内存/CPU/IO 瓶颈分析、低功耗策略（CPU idle、频率调控、外设电源域管理）。
- 自动化测试与 CI/CD：单元/集成测试、端到端自动化（pytest/Robot）、固件签名与流水线、自动化回归测试、覆盖率/静态分析工具（clang-tidy、cppcheck）。
- OTA 与版本管理：差分升级、回滚策略、升级完整性校验、版本兼容性策略。
- 安全与可靠性：输入校验、权限隔离、漏洞复现（fuzzing）、安全补丁发布流程。
- 客户支持流程（软件层面）：问题重现流程、必要日志清单、快速定位模板、补丁交付规范、SDK 兼容性表。

6 个月（24 周）阶段性计划与可量化交付

第1月（第1-4周）：环境与工具链、Boot 与日志

- 内容：搭建交叉编译环境，熟悉构建系统；理解 bootloader 与 boot 流程，能读取并分析 boot log；实践烧写固件并恢复出厂。学习日志策略与串口/网络日志收集。
- 考核目标：完成交叉工具链配置并构建一个最小 Linux 镜像（或固件）；能用串口获取完整 boot log 并定位启动失败的一处原因。
- 可量化标准：提交构建脚本/流水线（至少一次自动化构建）、boot log 分析报告（含问题与定位步骤）。

第2月（第5-8周）：BSP/驱动基本开发与 Device Tree

- 内容：编写或移植简单内核驱动（字符设备或 SPI/I2C 设备），理解 Device Tree 的节点/属性，完成驱动与设备树的绑定与测试。
- 考核目标：完成一个内核模块，实现用户态通过 ioctl/read/write 与设备交互；Patch 提交到代码仓库并记录编译/加载/测试步骤。
- 可量化标准：仓库历史不少于 6 次提交；README 包含加载/测试步骤；提供用户态示例程序与运行日志。

第3月（第9-12周）：系统集成、SDK 与示例应用

- 内容：设计并实现 SDK 层 API（C/Python），打包示例应用，完善 API 文档与示例；实现固件升级接口（本地/远程）。
- 考核目标：完成 SDK（含至少 3 个 API）并提供示例，能通过 SDK 调用完成设备功能；实现一次完整 OTA 更新测试并能回滚。
- 可量化标准：SDK 仓库、文档至少 3 页、OTA 测试报告（上传/下载/校验/回滚日志）。

第4月（第13-16周）：调试、profiling 与稳定性测试

- 内容：掌握 perf/ftrace/valgrind 等工具的使用；定位内存泄漏、竞态与死锁；进行长期稳定运行测试并收集日志/崩溃信息。
- 考核目标：定位并修复一个内存泄漏或竞态问题；提供性能基准（如吞吐量/响应时间）并给出优化建议。
- 可量化标准：提交分析报告（包含 flamegraph/trace）、修复补丁与回归验证日志。

第5月（第17-20周）：自动化测试、CI/CD 与回归

- 内容：搭建 CI 流水线（GitHub Actions/GitLab CI/Jenkins），实现自动化构建、签名、部署与回归测试；编写端到端自动化测试脚本。
- 考核目标：实现一次可复现的 CI 流程，包含构建->单元测试->打包->部署；自动回归用例覆盖关键功能（至少 10 个测试点）。
- 可量化标准：CI 报告、测试覆盖率数据、回归测试结果与流水线脚本。

第6月（第21-24周）：安全/稳定性/客户 SDK 支持与结业项目

- 内容：安全加固（fuzzing、静态分析）、制定补丁与发布流程；模拟客户集成场景并提供 SDK 支持；整合结业项目。
- 考核目标：完成一例客户集成案例（含定位、补丁、回归验证）、并完成结业项目：从构建、驱动、SDK 到自动化测试的整套交付。
- 可量化标准：提交客户支持记录、补丁 (patch)、测试报告、结业白皮书与演示材料。

  - 任务：学习如何把 kernel/mmc/ufs 驱动日志与物理操作对应，练习在驱动层查找日志来源（源码阅读片段）。
